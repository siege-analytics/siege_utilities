# Examples Directory Status

The top-level `examples/` directory is kept for legacy/demo artifacts and documentation notes.

Current policy:
- Maintained runnable examples should live with the package source (for example, `siege_utilities/examples/` or `siege_utilities/reporting/examples/`).
- Notebook-derived outputs intended for review visibility belong in `notebooks/output/` and remain tracked.
- Any one-off exploratory assets in `examples/` should be marked as archived and should not be referenced from primary user documentation.

For active examples, start with:
- `siege_utilities/examples/enhanced_features_demo.py`
- `siege_utilities/reporting/examples/comprehensive_mapping_example.py`
- `siege_utilities/reporting/examples/bivariate_choropleth_example.py`
